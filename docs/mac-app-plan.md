# macOS アプリ化メモ

## 目的

`qwen-tts-jp-starter` を、ターミナル操作なしで起動しやすい macOS アプリに近づける。

## 先に結論

現状の構成でも `.dmg` 化はできます。  
ただし、すぐにきれいな単体アプリになるわけではなく、次のどちらかの方針を選ぶのが現実的です。

## 方針A: Python と依存関係をまとめて `.app` 化する

- 候補: `py2app` または `Briefcase`
- メリット: ユーザーは `.app` を開くだけで使いやすい
- デメリット: `torch` と `qwen-tts` が重く、ビルドや配布サイズが大きくなる

## 方針B: 起動ランチャーだけ `.app` 化して、中では今の Python 環境を使う

- 候補: `Platypus`、`Automator`、薄いネイティブラッパー
- メリット: 先に作りやすい
- デメリット: 完全な単体アプリではなく、初回セットアップの導線がまだ必要

## このリポジトリでおすすめの順番

1. まずはランチャー型の `.app` を作る
2. 次に Python 同梱型の `.app` を試す
3. 最後に `.dmg` を配布用に整える

## なぜこの順番か

- いきなり完全同梱型にすると、`torch` とモデル依存で詰まりやすい
- 先に「ダブルクリックで起動できる」体験を作る方が、初心者向けの価値が早く出る
- そのあとで配布サイズや署名、`Gatekeeper` 対応を詰める方が安全

## 次にやる実装候補

- `Launcher.app` 用のシェルスクリプトを作る
- 初回セットアップの進捗を GUI で見せる
- アプリ内に「初回セットアップ」「起動」「更新」をまとめる
- 配布用の `.dmg` を作るビルドスクリプトを追加する
